# Контрольные вопросы

## 1. Чем отличается фиксированный и переменный шаги игрового цикла?

Фиксированный шаг использует постоянный интервал обновления (например, `1/60` сек), что удобно для физики и повторяемости поведения.
Переменный шаг использует фактическое время между кадрами (`deltaTime`), что проще реализовать, но требует аккуратной обработки больших скачков времени.

## 2. Зачем необходим RenderAdapter и можно ли обходиться без него?

`RenderAdapter` отделяет игровую логику от конкретной графической библиотеки.
Без него можно обойтись в маленьком проекте, но код становится сильно связанным с выбранным API и хуже расширяется.

## 3. Какова роль deltaTime в управлении физическими процессами и анимациями?

`deltaTime` позволяет масштабировать изменения по времени, а не по количеству кадров.
Из-за этого скорость движения и анимации остается примерно одинаковой при разном FPS.

## 4. Какие преимущества дает использование системы состояний в играх?

Система состояний разделяет поведение приложения по режимам (`загрузка`, `меню`, `игра`) и уменьшает количество условных веток в одном месте.
Это упрощает поддержку, тестирование и добавление новых режимов.

## 5. Как технически организовать смену графического API («горячую загрузку»)?

Нужно работать через абстракцию (`IRenderAdapter`) и отделить ресурсы от логики игры.
При смене API создается новый адаптер, переинициализируются графические ресурсы (шейдеры, буферы, текстуры), а логика игры продолжает работать через тот же интерфейс.
