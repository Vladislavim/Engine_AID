# Заметки для отчета

## Что написать в разделе "Общая архитектура"

Коротко описать:
- `Application` как центральный управляющий класс;
- систему состояний (`Loading`, `Menu`, `Gameplay`);
- `RenderAdapter` как слой абстракции над `raylib`;
- `Logger` для фиксации событий и `deltaTime`;
- `InputController` для обработки ввода отдельно от логики приложения.

## Какие листинги кода вставить в отчет

1. Интерфейс адаптера:
- `src/render/IRenderAdapter.h`

2. Главный цикл и расчет `deltaTime`:
- `src/core/Application.cpp`

3. Базовый интерфейс состояния и реализация состояний:
- `src/core/GameState.h`
- `src/core/States.cpp`

4. Обработка ввода с учетом `deltaTime`:
- `src/game/InputController.cpp`

## Какие скриншоты приложить

1. Окно приложения в `Menu state`
2. Окно в `Gameplay state` с видимым примитивом
3. Демонстрация перемещения/масштаба/поворота (можно 1-2 кадра)
4. Консоль с логами (`deltaTime`, смена состояний)

## Что написать в выводах

Примерный смысл:
- собран минимальный каркас игрового движка;
- реализован главный цикл и `deltaTime`;
- показана польза паттерна `Adapter` и системы состояний;
- архитектура пригодна для расширения (новые состояния, другой рендерер).
